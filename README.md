# 📂 Contexto do repositório

Este repositório foi criado com o objetivo de realizar a refatoração e modernização de um projeto em grupo desenvolvido no primeiro semestre da graduação.

A versão original do SmartTravel foi construída como um projeto acadêmico inicial, com foco em funcionamento básico e validação de conceitos introdutórios de desenvolvimento web. No entanto, ao longo da evolução dos estudos, foram identificadas diversas oportunidades de melhoria em relação à organização do código, arquitetura, padronização e boas práticas de desenvolvimento.

Dessa forma, este repositório tem como finalidade transformar o projeto inicial em uma aplicação mais estruturada, escalável e alinhada com padrões modernos de desenvolvimento web, servindo também como parte do portfólio acadêmico e profissional.

---

# ⚠️ Estado inicial do projeto

Durante a análise da versão original do SmartTravel, foram identificados diversos pontos de melhoria que motivaram o processo de refatoração e modernização da aplicação:

## 🧱 Estrutura e organização
- Estrutura de pastas desorganizada  
- Mistura de HTML, CSS e JavaScript no mesmo arquivo  
- Falta de separação clara de responsabilidades  

## 🎨 Interface e layout
- Inconsistência no header entre diferentes páginas  
- Falta de padronização visual  
- Baixa responsividade em dispositivos móveis  

## 🧠 Arquitetura e código
- Ausência de arquitetura definida  
- Código repetido em diferentes partes da aplicação  
- Dificuldade de manutenção e escalabilidade  

## 📦 Dados e integração
- Uso de JSON Server como simulação de backend  
- Dependência de dados locais sem API estruturada  

---

# 🔧 Melhorias planejadas para o backend

Planejamento da nova arquitetura backend da aplicação, com foco em escalabilidade, organização e segurança.

## 🛠️ Stack planejada

- Node.js  
- Express.js  
- MySQL  
- JWT (autenticação)  
- Zod (validação de dados)  
- Middlewares de autenticação  
- Arquitetura em camadas (MVC / Services)  

## 🎯 Objetivos

- Estruturar uma API REST organizada  
- Implementar autenticação segura  
- Separar regras de negócio da interface  
- Migrar de JSON Server para banco relacional  
- Melhorar escalabilidade e manutenção  

---

# 🎨 Melhorias planejadas para o frontend

Modernização da interface com foco em componentização, responsividade e manutenção.

## 🛠️ Stack planejada

- Vue.js  
- TypeScript  
- Vue Router  
- Arquitetura baseada em componentes  
- CSS responsivo  
- Consumo de API REST  

## 🎯 Objetivos

- Separar corretamente HTML, CSS e JS  
- Criar componentes reutilizáveis  
- Padronizar layout e identidade visual  
- Melhorar responsividade (mobile-first)  
- Reduzir duplicação de código  
- Melhorar experiência do usuário  
- Facilitar manutenção e escalabilidade
